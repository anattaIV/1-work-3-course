npm init -y - инитиализация npm
npm i typescript --save-dev - установка TS
----------------
npx tsc - запуск компилятора(копирование проекта в dist)
node dist/index.js - проверка результата функции introduce
---------------------------------------------------------------------
1. JsDoc - если грубо, то это просто комментарии в коде и это не панацея. Условно, человек может написать какую то хрень и ты не поймешь что он написал или объяснил очень кратко. У каждого разное мышление. Легче просто задать рамки, а не объяснять как что работает пытаясь написать так, чтобы каждый понял. 
2. Инференс - это когда JS угадывает типы переменных или других элементов сам. Например let=5 - это number. Из за этого и могут возникать ошибки.
3. tsconfig.json как бы настраивает ТС. Например, можно назначить тип модулей, версию компиляции, строгость фильтра ТС, папки в которой будет сам ТС и итоговый вариант из ДЖС.
4. ? - делает параметр опцианальным и необязательным для заполнения.
5. interface предназначен для описания формы объекта или класса. type — более универсальный, позволяет объединять типы (union, intersection).
6. Чтобы валидировать тип во время выполнения, используют библиотеки: zod, yup, io-ts
